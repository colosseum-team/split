# Post-hackathon roadmap

## 30 days

- Stabilize MVP contract lifecycle with end-to-end tests.
- Complete backend auth and role APIs for production-like reliability.
- Add basic dispute intake flow and evidence attachments.
- Improve dashboard UX for both `customer` and `user`.
- Prepare pilot user interviews and collect first feedback.

## 60 days

- Introduce milestone-based escrow (multiple releases per contract).
- Add better risk controls and richer contract templates.
- Improve indexing and search performance for contract histories.
- Integrate notifications for contract state changes.
- Run closed beta with initial user cohort.

## 90 days

- Launch public beta with monitoring and support workflows.
- Add reputation primitives based on completed contracts.
- Expand payment rails for stable token options where relevant.
- Formalize dispute resolution policy and operator tooling.
- Define growth metrics and weekly shipping cadence.

## Product milestones

- M1: Reliable create -> fund -> accept -> submit -> approve flow.
- M2: Dispute-ready workflows and evidence management.
- M3: Repeat usage from early teams and measurable retention signals.

## Team execution principles

- Ship in small weekly increments with visible changelog.
- Keep on-chain scope minimal but meaningful and secure.
- Validate with real users before adding major complexity.
- Prioritize UX clarity for trust-critical financial actions.
