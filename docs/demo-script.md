# Demo Script

## Goal

Show a complete and understandable escrow flow for a freelance contract on Solana.

## Demo length

2 to 4 minutes.

## Pre-demo checklist

- App builds and runs locally.
- Wallet is connected in browser.
- Seed data or test accounts are prepared for both roles.
- One ready-to-run contract scenario is available.

## Storyline

### 1) Problem and value (15-20 sec)

- "Freelancers need guaranteed payouts and clients need controlled release."
- "split provides role-based escrow with transparent contract states."

### 2) Customer flow (50-70 sec)

- Login/connect wallet as `customer`.
- Create a contract with title, scope, amount, and deadline.
- Fund escrow and show status update in UI.
- Highlight that payment is not released until approval logic.

### 3) User flow (50-70 sec)

- Switch to `user`.
- Accept open contract.
- Submit work for review.
- Show contract state transition and timeline.

### 4) Resolution flow (30-45 sec)

- Return to `customer`.
- Approve submission and release payout.
- Show final status as completed.

### 5) Solana proof point (20-30 sec)

- Briefly show transaction/account evidence (explorer link, tx hash, or state view).
- Explain why Solana is used: low fees + fast finality + transparent state.

### 6) Forward path (15-20 sec)

- Mention immediate roadmap: dispute tooling, richer reputation, production hardening.

## Voiceover notes

- Keep language user-centric, not purely technical.
- Explain each state transition in one short sentence.
- Keep focus on "problem solved" and "why it matters."

## Recording checklist

- Clean browser tabs and test data names.
- Zoom level readable on laptop screens.
- No dead time; pre-open required pages.
- Ensure narration matches on-screen actions.

## Deliverables to include with submission

- Public demo video link.
- GitHub repo link.
- README with run instructions and architecture references.
